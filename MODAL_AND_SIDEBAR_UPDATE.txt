üéØ PROJECT OVERVIEW
================================================================================
A comprehensive AWS-style documentation website for MyIDEX HUB Standard Operating
Procedures (SOPs). Features 41 fully documented sections across 5 modules with
advanced UI components, image viewer, search, animations, and responsive design.

Main Stack:
- Angular 21+ with @if/@for control flow
- Standalone Components with Signal-based state management
- Tailwind CSS v3 for styling
- HttpClient for data loading
- Default change detection (NOT OnPush - was causing scroll/animation issues)
- TypeScript with strict typing

üîß SERVICES ARCHITECTURE
================================================================================

1. DocsDataService
   - Loads JSON data from: /public/assets/data/myidex-hub-sop-complete.json
   - Methods:
     * getDocsData(): Observable<DocsData>
     * Gets all modules and sections
   - Used by: docs.component.ts
   - Cache strategy: Single HTTP request on first load

2. ImageViewerService
   - Manages image zoom/pan state
   - Signals:
     * imageZoom: scale factor (1x to 4x, or -1 for fit-screen)
     * panX, panY: pan offset coordinates
   - Methods:
     * zoomIn/Out()
     * fitToScreen()
     * resetPan()
     * rotate() [Note: Removed from use - caused bugs]
   - Used by: docs.component.ts image modal

  Issue: User requested highlight when clicking section links
  Implementation: Added CSS @keyframes (shake + pulse-highlight)
  Added: currentAnchor signal to track which section is highlighted
  Result: ‚úÖ Sections now shake (0.5s) and pulse when clicked
  
PHASE 4: Image Rotation Bug (360¬∞ flip)
  Issue: When opening image modal, image appeared rotated/flipped
  Root Cause: imageZoom signal was -1 (fit-to-screen), causing scale(-1) transform
  Fix: Added ternary operator: imageZoom() === -1 ? 1 : imageZoom()
  Result: ‚úÖ Images display correctly in modal
